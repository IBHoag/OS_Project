OS Project — AI‑Generated Code Evaluation
This repository contains the full dataset, code, logs, and analysis for evaluating AI‑generated C programs across five OS domains: Processes, Signals, Resources, Files, and IPC.
All programs were generated by an LLM and tested using a standardized execution harness to measure correctness, behavior, and failure modes.

Repository Structure
OS_Project/
│
├── Proc/          # Process‑related tasks (fork, exec, zombies, orphans)
├── Sig/           # Signal handling tasks (SIGINT, SIGCHLD, handlers)
├── Res/           # Resource management tasks (memory, CPU, limits)
├── Files/         # File I/O tasks (open, read, write, permissions)
├── IPC/           # Inter‑process communication tasks (pipes, FIFOs)
│
├── logs/
│   ├── all_error_logs/   # Consolidated stderr logs
│   └── all_output_logs/  # Consolidated stdout logs
│
├── run_test.sh    # Execution harness used for every trial
└── results.csv    # Final dataset of outcomes and classifications

Each task folder contains:

task.c (AI‑generated code)
run_test.sh (execution script)
output.log (stdout)
error.log (stderr)

Testing Methodology
Every program was executed using a consistent harness:
Compile with gcc -Wall -Wextra -o task task.c
Run with a timeout to prevent hangs

Capture:
stdout → output.log
stderr → error.log

Record:
Compilation success/failure
Runtime behavior
Semantic correctness
Failure category

This produced a reproducible dataset across all OS domains.

Failure Categories
Each trial was classified into one of the following:

Syntactic Failure — does not compile
Semantic Failure — compiles but produces incorrect behavior
Abstraction Failure — misunderstanding of OS concepts
Logical Failure — flawed reasoning or incorrect control flow
Success — correct behavior and output
These labels appear in results.csv.

Consolidated Logs
All error.log and output.log files were collected into:

Code
logs/all_error_logs/
logs/all_output_logs/
These folders allow quick inspection of patterns across tasks.

How to Re‑Run a Task
From any task directory:

Code
chmod +x run_test.sh
./run_test.sh
This will regenerate output.log and error.log.

Purpose of the Project
This project evaluates the reliability of AI‑generated C code in operating‑system contexts.
It highlights:

Common failure patterns
Misunderstood OS abstractions
Differences between syntactic and semantic correctness
The gap between “compiles” and “works correctly”
The dataset supports research into LLM behavior, OS education, and automated program evaluation.
